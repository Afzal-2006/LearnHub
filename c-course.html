1. The Life of a C Program

C is a compiled language. Unlike Python or JS, your code is converted directly into machine instructions that the CPU understands.

The 4 Stages of Compilation

StageProcessOutput File
PreprocessingRemoves comments, expands macros (#define), and includes headers..i file
CompilationTranslates code into Assembly instructions for the specific CPU..s file
AssemblyConverts assembly code into binary Machine Code (Object code)..o or .obj
LinkingCombines object files with system libraries (like stdio.h)..exe or a.out

The Structure of a C File

#include <stdio.h>  // Preprocessor directive

int main() {         // Entry point of the program
    printf("Hardware Control starts here!\n");
    return 0;        // 0 means success
}

3. Mastering Pointers (The Heart of C)

Pointers are variables that store Memory Addresses. This allows you to manipulate data directly in the RAM without creating copies.

💡 Why use Pointers?
  • Dynamic Memory Allocation (Heap)
  • Efficiently passing large arrays to functions
  • Building data structures like Linked Lists and Trees
int x = 10;
int *p = &x;  // Pointer 'p' stores address of 'x'

printf("Address of x: %p\n", p);
printf("Value at that address: %d\n", *p); // Dereferencing

// Manual Memory Allocation
int *arr = (int*)malloc(5 * sizeof(int)); // Ask OS for 20 bytes
if(arr == NULL) return 1; // Check if allocation failed
free(arr); // ALWAYS clean up your memory!

4. Structures and Persistent Storage

Structs allow you to group different variables under one name. File I/O lets you save your data permanently to the hard drive.

Complex Data Grouping

struct Student {
    char name[50];
    int roll;
    float gpa;
};

struct Student s1 = {"Moin", 101, 3.8};
printf("Student %s has GPA %.2f", s1.name, s1.gpa);
