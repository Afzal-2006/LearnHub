1. Classes and Encapsulation

C++ is C "with Classes". Encapsulation is the practice of bundling data and the methods that act on that data into a single unit (Class) and hiding the implementation details from the outside world.

class Employee {
private:
    double salary;  // Data Hiding (Private)

public:
    string name;
    
    // Setter
    void setSalary(double s) {
        if(s > 0) salary = s;
    }
    
    // Getter
    double getSalary() {
        return salary;
    }
};

2. Inheritance & Access Control

Inheritance allows a "Child" class to inherit attributes and methods from a "Parent" class, promoting code reusability.

    void honk() { cout << "Tuut, tuut!" << endl; }

class Car : public Vehicle {

    string model = "Mustang";
};

3. Polymorphism & Virtual Functions

Polymorphism means "many forms". It allows objects of different classes to be treated as objects of a common base class through Virtual Functions.

💡 Runtime Polymorphism: Occurs when a child class overrides a parent class method using the virtual keyword.
class Shape {
public:
    virtual void area() = 0; // Pure Virtual Function (Abstract)
};

class Circle : public Shape {
public:
    void area() override { cout << "Area of Circle is PI*r^2" << endl; }
};

class Square : public Shape {
public:
    void area() override { cout << "Area of Square is a^2" << endl; }
};
