1. Classes, Objects, and Encapsulation

C++ extends C by adding classes, stronger abstraction, constructors, destructors, templates, and powerful ways to organize code. One of the most important ideas is encapsulation, which means combining data and the methods that operate on that data into a single unit.

Encapsulation improves maintainability because it helps you hide internal implementation details. This means other parts of the program interact with a class through a defined public interface instead of directly manipulating internal state.

Basic Class Example

class Employee {
private:
    double salary;

public:
    string name;

    void setSalary(double s) {
        if (s > 0) salary = s;
    }

    double getSalary() {
        return salary;
    }
};

Here, salary is private, which means external code cannot access it directly. Instead, the class provides methods to safely update and retrieve the value.

Why Encapsulation Matters

  • It protects object state.
  • It reduces accidental misuse.
  • It creates cleaner code boundaries.
  • It makes future changes easier.
💡 Best Practice: Keep internal data private unless there is a strong reason not to. Let public methods control how that data is used.

2. Inheritance and Access Control

Inheritance allows a child class to reuse properties and methods from a parent class. This reduces duplication and helps organize related concepts into hierarchies.

Access Modifiers

Modifier Inside Class Derived Class Outside Class
public Yes Yes Yes
protected Yes Yes No
private Yes No No

Inheritance Example

class Vehicle {
public:
    string brand = "Ford";

    void honk() {
        cout << "Tuut, tuut!" << endl;
    }
};

class Car : public Vehicle {
public:
    string model = "Mustang";
};

In this example, Car inherits the brand property and the honk() method from Vehicle.

When to Use Inheritance

Inheritance works best when a child is a specialized form of the parent. For example, a car is a type of vehicle. However, inheritance should be used carefully. Sometimes composition is cleaner than inheritance.

Important: Inheritance is powerful, but overusing it can create rigid class hierarchies. Always think about real relationships between objects.

3. Polymorphism and Virtual Functions

Polymorphism means “many forms.” In C++, it allows code to work through a common interface while different derived classes provide different implementations.

Virtual Functions

Virtual functions enable runtime polymorphism. That means the correct function is chosen based on the actual object type at runtime, not just the pointer type.

class Shape {
public:
    virtual void area() = 0;
};

class Circle : public Shape {
public:
    void area() override {
        cout << "Area of Circle is PI*r^2" << endl;
    }
};

class Square : public Shape {
public:
    void area() override {
        cout << "Area of Square is a^2" << endl;
    }
};

The function in Shape is a pure virtual function. That makes Shape an abstract class, which means you cannot create a direct object of it.

Why Polymorphism Is Useful

  • It allows flexible code design.
  • It supports interfaces and reusable architecture.
  • It helps different classes behave correctly under a shared base type.
💡 Tip: Polymorphism becomes especially valuable in larger systems, where many objects share a general contract but implement details differently.

4. Standard Template Library (STL)

The Standard Template Library is one of the most powerful features of C++. It provides reusable containers, iterators, and algorithms so you do not need to build every data structure from scratch.

Important Containers

Container Purpose Typical Benefit
std::vector Dynamic array Fast indexed access
std::map Key-value store Sorted lookup
std::list Doubly linked list Flexible insertion/deletion
std::queue FIFO structure Task ordering
std::stack LIFO structure Undo/history patterns

Vector and Algorithm Example

#include <vector>
#include <algorithm>

vector<int> v = {5, 2, 9, 1};
v.push_back(10);
sort(v.begin(), v.end());

for (int n : v) {
    cout << n << " ";
}

STL makes code shorter, safer, and more powerful. It also encourages using tested, standardized tools rather than rewriting common operations manually.

5. Memory Management, Constructors, and Advanced Thinking

C++ gives more control than many higher-level languages, but that also means memory and object lifecycle matter a lot. Constructors initialize objects, destructors clean them up, and dynamic memory introduces responsibility.

Constructors

class Student {
public:
    string name;
    int age;

    Student(string n, int a) {
        name = n;
        age = a;
    }
};

Destructor

class Demo {
public:
    Demo() {
        cout << "Object created" << endl;
    }

    ~Demo() {
        cout << "Object destroyed" << endl;
    }
};

Dynamic Memory

int *ptr = new int(10);
cout << *ptr << endl;
delete ptr;

Modern C++ often prefers safer alternatives such as smart pointers, but understanding raw new and delete is still important for learning the language deeply.

Remember: Good C++ developers think not only about syntax, but also about ownership, object lifetime, efficiency, and maintainability.

6. C++ Best Practices and Final Quiz

To write good C++ code, focus on clarity and correctness. Complex features do not automatically make code better. In many cases, disciplined design matters more than language tricks.

Best Practices

  • Keep classes focused and meaningful.
  • Hide internal state unless public access is necessary.
  • Use inheritance only when the relationship is truly logical.
  • Prefer STL containers and algorithms when possible.
  • Manage memory carefully and avoid leaks.
