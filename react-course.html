1. The Virtual DOM

React became popular because it made user interface development more predictable and component-based. One of its most important concepts is the Virtual DOM. Instead of directly updating the browser DOM every time data changes, React first creates a lightweight in-memory representation of the UI.

When state or props change, React compares the new virtual tree with the previous one. This process is called diffing. After diffing, React updates only the parts of the real DOM that actually changed. This overall strategy is called reconciliation.

Why React Uses a Virtual DOM

  • Direct DOM updates can be expensive.
  • React reduces unnecessary updates.
  • UI changes become more predictable.
  • Developers can think in terms of state, not manual DOM manipulation.
Important: The Virtual DOM is not a magical replacement for performance issues. It is a smart optimization layer that works well with React’s component model.

Core Idea

Instead of saying “change this exact DOM node manually,” you tell React what the UI should look like for the current data. React handles the DOM update details for you.

💡 Key Idea: In React, UI is treated like a function of state. Change the state, and React updates the UI accordingly.

2. JSX: JavaScript XML

JSX lets developers write HTML-like syntax inside JavaScript. This makes UI code easier to read because structure and logic stay close together. Even though JSX looks like HTML, browsers do not understand it directly. Build tools transform JSX into regular JavaScript function calls.

Basic JSX Example

const name = "LearnHub";
const element = <h1>Welcome to {name}</h1>;

The expression inside curly braces is JavaScript. That means JSX is flexible because it can combine markup-like syntax with actual logic.

Why JSX Helps

  • UI becomes easier to read.
  • Logic and structure stay close together.
  • Components become clearer and more maintainable.

JSX Rules

  • Return one parent element.
  • Use className instead of class.
  • JavaScript expressions go inside curly braces.
  • Every tag must be closed properly.
function App() {
  return (
    <div className="container">
      <h2>React UI</h2>
      <p>Learning JSX</p>
    </div>
  );
}

3. Functional Components and Props

React applications are built from components. A component is a reusable piece of UI. Instead of building one huge page file, React encourages breaking the interface into smaller, focused parts.

Functional Components

function UserProfile() {
  return (
    <div>
      <h2>User Info</h2>
      <p>Active Student</p>
    </div>
  );
}

Functional components are simply JavaScript functions that return JSX. They are now the main standard in modern React development.

Props

Props are read-only inputs passed from parent components to child components. They let one component become dynamic.

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Usage:
// <Greeting name="Alex" />
  • Props flow from parent to child.
  • Props are read-only.
  • They help components become reusable.
💡 Component Thinking: Build UIs like Lego blocks. Small reusable components are easier to test, style, and maintain.

4. Hooks: useState and useEffect

Hooks let functional components use state and lifecycle-related features. Before hooks, many of these capabilities required class components. Now, hooks make functional components much more powerful.

useState

useState lets a component remember data between renders.

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

useEffect

useEffect is used for side effects such as API calls, subscriptions, timers, or manual browser interactions.

import { useEffect } from 'react';

useEffect(() => {
  console.log("Component Mounted!");
  return () => console.log("Unmounting...");
}, []);

The empty dependency array means the effect runs once after the first render.

Important: Hooks must be called at the top level of a React component, not inside loops or conditions.

5. State, Re-rendering, Lists, and UI Flow

One of the biggest strengths of React is the relationship between state and rendering. When state changes, React re-renders the component so the UI stays in sync with the data.

Rendering Lists

React commonly renders lists using map(). Each list item needs a unique key so React can track updates efficiently.

const items = ['React', 'Vue', 'Angular'];

return (
  <ul>
    {items.map(item => <li key={item}>{item}</li>)}
  </ul>
);

Re-rendering

React re-renders when props or state change. This is not the same as redrawing the whole page blindly. React uses reconciliation to update only the parts that changed.

Custom Hooks and Code Reuse

Custom hooks let you reuse logic across components without repeating code.

function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);

  return { count, increment, decrement };
}
  • State controls what the UI shows.
  • Props allow data to move between components.
  • Hooks allow reusable logic.
  • Keys help React manage list updates efficiently.

6. React Best Practices and Final Quiz

Strong React development is not about memorizing hook names only. It is about thinking in components, keeping state manageable, and making the UI predictable.

Best Practices

  • Break large UIs into smaller reusable components.
  • Use props clearly and keep them meaningful.
  • Keep state as local as possible.
  • Use keys correctly when rendering lists.
  • Write hooks only where they improve clarity and reuse.
