1. The Virtual DOM

React doesn't update the real browser DOM directly because it's slow. Instead, it creates a Virtual DOM—a lightweight copy in memory.

When state changes, React compares the new Virtual DOM with the old one (Diffing) and only updates the specific parts of the real DOM that changed (Reconciliation).

2. JSX: JavaScript XML

JSX allows us to write HTML-like code inside JavaScript. It makes components more readable.

const name = "LearnHub";
const element = <h1>Welcome to {name}</h1>;

Note: Browsers can't read JSX directly; it is compiled into React.createElement() by Babel.

3. Functional Components

Components are the building blocks of React. Functional components are simply JavaScript functions that return JSX.

function UserProfile() {
  return (
    <div>
      <h2>User Info</h2>
      <p>Active Student</p>
    </div>
  );
}

4. Understanding Props

Props (Properties) are read-only inputs passed from a parent component to a child. They allow components to be dynamic.

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Usage: <Greeting name="Alex" />

5. The useState Hook

State allows components to "remember" data. When state updates, React re-renders the component.

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}

6. The useEffect Hook

Used for "side effects" like fetching data, manual DOM changes, or setting up subscriptions.

useEffect(() => {
  console.log("Component Mounted!");
  return () => console.log("Unmounting...");
}, []); // Empty array means run once

7. Lists and Keys

We use .map() to render lists. Every list item needs a unique key prop so React can track it efficiently.

const items = ['React', 'Vue', 'Angular'];
return (
  <ul>
    {items.map(item => <li key={item}>{item}</li>)}
  </ul>
);

8. Final Project Structure

In a pro environment, we organize components into folders. We use Custom Hooks to separate logic from UI.
