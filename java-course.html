☕ Java Basics

Java is a powerful, object-oriented programming language. It follows the principle "Write Once, Run Anywhere" (WORA), making it ideal for cross-platform development.

What is Java?

Java is compiled to bytecode that runs on the Java Virtual Machine (JVM), not on the operating system directly. This makes Java programs portable across different operating systems.

Your First Java Program

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
Breaking it down:
  • public class HelloWorld - Class declaration (must match filename)
  • public static void main - Main method (entry point)
  • String[] args - Command line arguments
  • System.out.println() - Print to console

How to Run Java Programs

// Compile the Java file
javac HelloWorld.java

// Run the compiled program
java HelloWorld

// Output: Hello, World!
💡 Key Point: Java is case-sensitive! System is different from system.

📦 Variables & Data Types

Variables store data values. Java is a strongly-typed language, meaning every variable must have a declared data type.

Primitive Data Types

Type Size Range Example
byte 1 byte -128 to 127 byte b = 100;
int 4 bytes -2.1B to 2.1B int age = 25;
long 8 bytes Very large numbers long id = 123456789L;
float 4 bytes Decimal numbers float price = 19.99f;
double 8 bytes Decimal numbers (precise) double salary = 50000.50;
boolean 1 bit true or false boolean isActive = true;
char 2 bytes Single character char grade = 'A';
String Variable Text String name = "John";

Declaring and Using Variables

// Variable declaration and initialization
int count = 10;
double price = 29.99;
String name = "Alice";
boolean isStudent = true;

// Using variables
System.out.println("Name: " + name);
System.out.println("Price: $" + price);
System.out.println("Count: " + count);

// Changing variables
count = count + 5;  // count is now 15
price = 39.99;      // updated price

Type Conversion

// Implicit conversion (automatic)
int num = 42;
double decimal = num;  // int → double (safe)
System.out.println(decimal);  // 42.0

// Explicit conversion (casting)
double pi = 3.14159;
int rounded = (int) pi;  // double → int
System.out.println(rounded);  // 3

// String to number
String str = "123";
int value = Integer.parseInt(str);  // "123" → 123
double d = Double.parseDouble("3.14");  // "3.14" → 3.14
💡 Pro Tip: Use final keyword to create constants that cannot be changed.

🏗️ Object-Oriented Programming

Java is built on OOP principles. Classes are blueprints for creating objects, and objects contain data (properties) and behavior (methods).

Classes and Objects

// Define a class
public class Car {
    // Properties (attributes)
    String color;
    String brand;
    int year;
    
    // Constructor
    public Car(String color, String brand, int year) {
        this.color = color;
        this.brand = brand;
        this.year = year;
    }
    
    // Methods
    public void displayInfo() {
        System.out.println("Car: " + brand + " " + color + " (" + year + ")");
    }
}

// Using the class
public class Main {
    public static void main(String[] args) {
        // Create objects
        Car car1 = new Car("Red", "Ferrari", 2023);
        Car car2 = new Car("Blue", "Tesla", 2024);
        
        // Use methods
        car1.displayInfo();  // Output: Car: Ferrari Red (2023)
        car2.displayInfo();  // Output: Car: Tesla Blue (2024)
    }
}

Encapsulation - Getters & Setters

public class Student {
    // Private properties
    private String name;
    private int age;
    private double gpa;
    
    // Getter methods
    public String getName() {
        return name;
    }
    
    public int getAge() {
        return age;
    }
    
    // Setter methods
    public void setName(String name) {
        if (name != null && !name.isEmpty()) {
            this.name = name;
        }
    }
    
    public void setAge(int age) {
        if (age > 0 && age < 120) {
            this.age = age;
        }
    }
    
    public void setGPA(double gpa) {
        if (gpa >= 0.0 && gpa <= 4.0) {
            this.gpa = gpa;
        }
    }
}

Inheritance

// Parent class
public class Animal {
    protected String name;
    
    public Animal(String name) {
        this.name = name;
    }
    
    public void makeSound() {
        System.out.println("Some generic animal sound");
    }
}

// Child class
public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " says: Woof! Woof!");
    }
}

// Using inheritance
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog("Buddy");
        dog.makeSound();  // Output: Buddy says: Woof! Woof!
    }
}

⚙️ Methods

Methods are functions that belong to a class. They perform specific tasks and can return values.

Method Syntax

// Method structure
public returnType methodName(parameterType parameter) {
    // Method body
    return value;
}

// Example
public int add(int a, int b) {
    int sum = a + b;
    return sum;
}

Different Types of Methods

public class Calculator {
    // Method with parameters and return value
    public int multiply(int x, int y) {
        return x * y;
    }
    
    // Method with no parameters and no return value
    public void printMessage() {
        System.out.println("Hello from method!");
    }
    
    // Method with multiple parameters
    public double calculateAverage(double a, double b, double c) {
        return (a + b + c) / 3;
    }
    
    // Method with no parameters but return value
    public String getCurrentTime() {
        return java.time.LocalTime.now().toString();
    }
}

// Using the methods
public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        System.out.println("5 × 3 = " + calc.multiply(5, 3));
        calc.printMessage();
        System.out.println("Average: " + calc.calculateAverage(80, 85, 90));
        System.out.println("Time: " + calc.getCurrentTime());
    }
}

Method Overloading

// Same method name, different parameters
public class MathOperations {
    // Method 1: Adds two integers
    public int add(int a, int b) {
        return a + b;
    }
    
    // Method 2: Adds three integers
    public int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Method 3: Adds two doubles
    public double add(double a, double b) {
        return a + b;
    }
    
    // Method 4: Concatenates strings
    public String add(String a, String b) {
        return a + b;
    }
}

// Using overloaded methods
MathOperations math = new MathOperations();
System.out.println(math.add(5, 10));           // 15
System.out.println(math.add(5, 10, 15));       // 30
System.out.println(math.add(3.5, 2.5));        // 6.0
System.out.println(math.add("Hello", "World")); // HelloWorld
💡 Key Points:
  • Use void when method doesn't return anything
  • Method names should describe what they do
  • Methods can be reused multiple times
  • Overloading lets you use same method name with different parameters
