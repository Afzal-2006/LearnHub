1. Java Basics and Program Structure

Java is a powerful object-oriented programming language known for portability, reliability, and strong structure. It follows the principle Write Once, Run Anywhere, meaning Java code is compiled into bytecode and executed on the Java Virtual Machine (JVM) instead of running directly on the operating system.

This design makes Java an excellent language for education, enterprise software, backend systems, and Android development. It enforces structure more strongly than many scripting languages, which helps students learn disciplined programming habits.

Basic Java Program

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

In this example:

  • public class HelloWorld defines the class.
  • main() is the entry point of the program.
  • System.out.println() prints text to the console.

Why Java Is Structured

Java forces code into classes because it is designed around object-oriented principles. This encourages developers to think in terms of reusable units rather than loose statements scattered across files.

💡 Best Practice: Java is case-sensitive. Even small changes in capitalization can break your code, so always type class names and method names carefully.

How Java Runs

javac HelloWorld.java
java HelloWorld

The first command compiles the source file into bytecode. The second command runs that bytecode using the JVM.

2. Variables, Data Types, and Type Conversion

Java is a strongly typed language. Every variable must have a declared type. This improves safety because the compiler can detect many mistakes before the program even runs.

Common Data Types

Example Variable Usage

int count = 10;

System.out.println("Name: " + name);
System.out.println("Price: $" + price);

Type Conversion

Java allows automatic widening conversions, but narrowing conversions require explicit casting.

int num = 42;
double decimal = num; // automatic

double pi = 3.14159;
int rounded = (int) pi; // explicit cast
Important: Explicit casting can lose information. For example, converting a double to an int removes the decimal part.

3. Object-Oriented Programming Concepts

Java is built on object-oriented programming. OOP organizes programs into classes and objects, which improves modularity, reuse, and maintainability.

Classes and Objects

public class Car {
    String color;
    String brand;
    int year;

    public Car(String color, String brand, int year) {
        this.color = color;
        this.brand = brand;
        this.year = year;
    }

    public void displayInfo() {
        System.out.println("Car: " + brand + " " + color + " (" + year + ")");
    }
}

A class is a blueprint. An object is a real instance of that blueprint.

Encapsulation

Encapsulation protects internal data by using private fields and public getters/setters.

public class Student {
    private String name;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        if (name != null && !name.isEmpty()) {
            this.name = name;
        }
    }
}

Inheritance

Inheritance allows a class to reuse and extend another class.

class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }
}

class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    public void bark() {
        System.out.println(name + " says: Woof!");
    }
}
💡 OOP Idea: OOP is useful because it lets you model real-world entities using clear structure and reusable logic.

4. Methods, Parameters, and Overloading

Methods are functions that belong to a class. They define behavior and make code reusable. A good method has a clear name, a focused responsibility, and meaningful parameters.

Method Example

public int add(int a, int b) {
    return a + b;
}

Methods with Different Forms

public void printMessage() {
    System.out.println("Hello from method!");
}

public double calculateAverage(double a, double b, double c) {
    return (a + b + c) / 3;
}

Method Overloading

Java allows multiple methods with the same name as long as their parameter lists differ.

public int add(int a, int b) {
    return a + b;
}

public int add(int a, int b, int c) {
    return a + b + c;
}

public double add(double a, double b) {
    return a + b;
}

Overloading improves readability because one concept can be expressed with one method name across related cases.

5. Control Flow, Arrays, and Practical Logic

Strong programming depends on more than syntax. You also need to understand decision-making, repetition, and grouped data handling.

Conditionals

int age = 20;

if (age >= 18) {
    System.out.println("Adult");
} else {
    System.out.println("Minor");
}

Loops

for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

Arrays

Arrays store multiple values of the same type.

int[] numbers = {10, 20, 30, 40};

for (int i = 0; i < numbers.length; i++) {
    System.out.println(numbers[i]);
}

Enhanced For Loop

for (int num : numbers) {
    System.out.println(num);
}
Remember: Good logic comes from understanding program flow clearly. Always think about conditions, loop boundaries, and what data structure fits the task best.

6. Java Best Practices and Final Quiz

Strong Java code is clear, organized, and maintainable. Java rewards developers who write descriptive methods, keep classes focused, and respect object-oriented design principles.

Best Practices

  • Use meaningful variable and method names.
  • Keep classes focused on one responsibility.
  • Use encapsulation to protect internal state.
  • Prefer readable logic over unnecessarily clever code.
  • Test methods independently when possible.
