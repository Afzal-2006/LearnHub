1. Big O Notation

Big O describes the efficiency of an algorithm. It measures how time or space requirements grow as the input size (n) increases.

  • O(1): Constant time (Fastest)
  • O(log n): Logarithmic time (Binary Search)
  • O(n): Linear time (Loops)
  • O(n²): Quadratic time (Nested loops)

2. Array Logic

Arrays are stored in contiguous memory. This makes accessing an element by index extremely fast: O(1).

However, inserting or deleting at the start of an array is O(n) because every other element must be shifted.

3. Linked Lists

Unlike arrays, Linked Lists are nodes scattered in memory, connected by pointers.

class Node {
  constructor(data) {
    this.data = data;
    this.next = null;
  }
}

Benefit: Fast insertion/deletion at any point if you have the pointer.

4. Stacks (LIFO)

Stacks follow Last-In, First-Out. Common operations are push() and pop().

Used in: Undo buttons, function calls (the "Call Stack"), and reversing data.

5. Queues (FIFO)

Queues follow First-In, First-Out. Common operations are enqueue() and dequeue().

Used in: Printer queues, task scheduling, and Breadth-First Search.

6. Hash Tables

A Hash Table stores key-value pairs. It uses a Hash Function to turn a key into an index in an array.

Average lookup time is O(1), making it one of the most powerful structures in programming.

7. Binary Search Trees (BST)

A Tree where each node has at most two children. For any node, left children are smaller, and right children are larger.

Search complexity: O(log n).

8. Basic Sorting

Sorting is the process of arranging data in order. Bubble Sort is O(n²), while Quick Sort is O(n log n).
